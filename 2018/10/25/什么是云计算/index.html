什么是云计算

1 现实中的例子

假如你有一个很好的创业点子，但是为了实现它，需要商业应用程序，开发商业应用程序十分昂贵，少则十万，多则百万。每一个应用程序背后都有复杂的软硬件支持，它们需要数据中心、机房、电力、冷却系统、带宽、网络、服务器、存储空间和复杂的软件等等，此外还需要一批专业的专家团队安装、配置、运行和维护它们。总的来说需要开发、测试、组织、生产、维护等缺一不可的步骤，如果其中任意一步出现问题了，解决起来是很麻烦的。当软件版本更新时，还可能导致整个系统瘫痪，又要花费很多人力和财力去修复故障。一般的小型创业公司是无法承担的起这样昂贵的费用。

2 云计算的出现

云计算的出现改变了这一现状，云计算服务可以使小型创业公司不用自己运行应用，而是在一个共享的数据中心运行，就像 Gmail 和 Mircosoft Exchange Server 的对比，用户使用 Gmail 不需要服务器和存储空间，用户使用 Gmail 不需要专业团队去维护它，用户使用 Gmail 也不需要人工升级系统，当用户使用任何云端的应用时，只需要登陆，配置并开始使用，这就是云计算的力量。

云计算的出现改变了人们看待软件的方式，这种模型不仅仅用于消费者软件，还适用于商业软件，我们称为商用云计算。客户可以在云端运行着各种各样的应用程序，包括定制应用程序，因为它们可以在很短的时间内就能准备好应用程序所需的各类软硬件设施，客户不需要为运行所需的人员、产品、设备而付费，而且它们比大部分的应用程序更加可扩展、更安全、更可靠。

之所以云计算能够颠覆传统软件模式是因为云计算都基于一种多重租凭的基础设施，多重租凭应用不是每个公司有一个应用程序副本， 而是所有人共享一个应用程序，但它非常灵活，足以满足每个人的个性化需求，它就像一个巨大的写字楼，每个人共享其基础设施和服务，比如保洁服务，安保服务，水电服务和饮食服务等，但每个公司都能拥有个性化的办公空间，这意味着应用程序是具有弹性的，应用程序可以扩大到数万用户或者缩小到几个用户。

在收费方面，云计算服务提供商往往根据客户使用了多少资源来计费，客户对资源的使用可以做到即用即弃，而不必担心买回来闲置。

3 云计算的服务模式

3.1 基础设施及服务 IaaS

基础设施及服务（Infrastructure as a Service，IaaS）在三层服务架构的最底层，它通过虚拟化技术将服务器等计算平台、存储和网络资源打包，通过 API 接口的方式提供给用户，这样客户就不用自己购买机房和维护服务器。常见的 IaaS 服务有 Amazon EC2。

购买 IaaS 服务就好比购买生鲜食材自己做饭，以包饺子为例，买的面粉需要自己和面擀皮，买的肉需要自己剁碎，买的蔬菜需要自己剁碎，饺子需要自己包，最后煮熟了就能吃到饺子了。

3.2 平台及服务 PaaS

平台及服务（Platform as a Service，PaaS）运行在 IaaS 之上，它除了基础架构之外还提供了软件应用的开发组件和运行环境，通常还具备应用接口和中间键，一般使用 IaaS 和 PaaS 这两种服务的人都需要有一定技术背景。常见的 PaaS 服务有阿里云、腾讯云等等。

购买 PaaS 服务就好比直接购买速冻饺子，需要自己煮，直接煮熟了就能吃了。

3.3 软件及服务 SaaS

软件及服务（Software as a Service，SaaS）运行在三层服务架构的最顶层，是直接面向用户的应用层，SaaS 服务也是最成熟和知名度最高的服务模式，SaaS 服务就是把服务器放在云端，用户只需要有一台能够上网的设备就能直接通过 Web 浏览器或者 APP 来使用，用户不需要维护服务器也不需要技术人员的支持。常见的 SaaS 服务有 Gmail、QQ等等。

购买 SaaS 服务就好比直接点外卖，外卖到了就可以吃了。
